# 深度总结：《构建 Claude Code 的经验：我们如何使用技能（Skills）》

---

## 一、核心主题

本文是 Anthropic 官方团队基于其在内部大规模使用 Claude Code Skills（技能）的实践经验所撰写的深度总结。文章的核心主题是：**如何系统化地设计、分类和使用 Claude Code 中的"技能"（Skills）功能，以最大化 AI 辅助开发的效率与质量**。这不是一篇理论性介绍，而是来自数百个内部技能在实际工程环境中运行后沉淀出的第一手经验。

---

## 二、主要内容

### 什么是技能（Skills）？

文章首先澄清了一个常见误解：Skills **不只是 Markdown 文件**，而是包含脚本、资产、数据等多种资源的**文件夹结构**，AI 代理（Agent）可以发现、探索并操作其中的内容。在 Claude Code 中，Skills 还支持丰富的配置选项，包括注册动态钩子（Dynamic Hooks）。Anthropic 发现，充分利用配置选项和文件夹结构的技能，往往是效果最显著的。

---

### 技能的九大分类

Anthropic 对内部全部 Skills 进行了系统梳理，归纳出以下九个核心类别。这一框架不仅是对已有实践的总结，也为团队识别技能库中的空白提供了工具：

**1. 库与 API 参考类（Library and API Reference）**
用于解释如何正确使用内部库、CLI 工具或 SDK。通常包含参考代码片段和常见"坑点"（Gotchas）列表，帮助 Claude 规避错误。例如：内部计费库的边界情况说明、内部 CLI 的子命令文档等。

**2. 产品验证类（Product Verification）**
描述如何测试和验证代码是否正常运行，常与 Playwright、tmux 等外部工具配合使用。**这是对 Claude 输出质量影响最显著的技能类别**。文章建议可以让工程师专门花一周时间来优化验证技能，甚至采用让 Claude 录制输出视频、在每个步骤进行程序化断言等高级手段。

**3. 数据获取与分析类（Data Fetching and Analysis）**
连接团队的数据和监控系统，包含数据库查询模式、凭证配置、仪表盘 ID 等。例如：用户漏斗查询技能、Grafana 数据源配置、Datadog 字段参考等。

**4. 业务流程与团队自动化类（Business Process and Team Automation）**
将重复性工作流程自动化为单一命令。通常指令较为简单，但可能依赖其他技能或 MCP（Model Context Protocol）工具。建议将历史执行结果保存到日志文件，帮助模型保持一致性。例如：自动生成每日站会汇报、创建工单并触发后续工作流等。

**5. 代码脚手架与模板类（Code Scaffolding and Templates）**
为特定功能生成代码框架。当脚手架有自然语言层面的要求、纯代码无法完全表达时，这类技能尤为有价值。例如：生成新服务/工作流的标准模板、迁移文件模板等。

## 四、实际意义

### 对开发团队的价值

1. **方法论层面**：文章提供了一套可直接落地的技能分类框架，团队可用它审视自己的 Skills 库，快速发现未覆盖的场景。

2. **质量提升**：产品验证类技能被着重强调，其中提到的"视频录制输出"和"逐步状态断言"等做法，代表了将 AI 代理的输出质量从"猜测"提升到"可验证"的关键思路。

3. **规模化运营**：当一个团队需要管理数百个技能时，系统化分类不仅是整理工具，更是治理框架。好的分类有助于技能复用、降低重复建设，并让新成员快速上手。

4. **设计原则指引**：文章反复强调"单一职责"原则——一个技能只做一件事。这与软件工程中的模块化思想一脉相承，避免"大而全"的技能变成 AI 的负担。

### 对 AI 工程化实践的启示

这篇文章的深层意义在于：它揭示了**AI 辅助开发不是简单地"写 Prompt 然后等结果"**，而是需要围绕 AI 代理构建一套完整的"工具生态"——包括参考文档、验证机制、数据接口、流程自动化等。Skills 体系本质上是在为 AI 代理构建一个结构化的"工作上下文"，让其能够在特定的组织环境中高效、准确地完成任务。

Anthropic 的这套实践表明，未来企业级 AI 工程的核心竞争力之一，将是**如何高质量地构建和维护 AI 代理的技能体系**——这和过去积累代码库、文档库的逻辑高度相似，但面向的服务对象从人类工程师变成了 AI 代理本身。

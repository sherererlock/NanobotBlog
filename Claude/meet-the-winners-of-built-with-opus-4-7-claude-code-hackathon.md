# Built with Opus 4.7 Claude Code Hackathon 获奖项目深度总结

## 核心主题

本文介绍了 Anthropic 举办的"Built with Opus 4.7 Claude Code Hackathon"线上黑客松的获奖项目。参赛者借助 Claude Opus 4.7 模型及 Claude Code 工具，在短短数天内独立构建出覆盖医疗培训、电子维修、编程教育、家庭维修及工厂维护等多个垂直领域的创新应用。这场比赛充分展示了 AI 辅助开发的巨大潜力，以及普通开发者甚至非科班出身的建造者，如何借助先进 AI 工具实现过去需要大型团队才能完成的工程目标。

---

## 主要内容

### 第一名：Medkit — Bedirhan Keskin（土耳其·伊斯坦布尔）

**项目背景与痛点**
Bedirhan Keskin 是一位由执业医师转型的软件工程师。他亲身经历过年轻医生在急诊室独立面对大量陌生病例时的无助感——因为医学院的训练往往无法覆盖所有真实场景，实习医生往往被迫在真实患者身上"边学边练"。

**项目功能**
Medkit 利用 Claude Managed Agents 构建了一个**游戏化的模拟医疗诊所**，专为医学住院医师和初级医生设计。用户可以在虚拟环境中完整模拟问诊流程——采集病史、开具检验单、阅读影像资料、做出诊断、制定治疗方案。全程结束后，系统中的 AI 评估代理会按照真实临床指南（与医学委员会考试标准一致）对整个诊疗过程进行评分。

**技术实现**
Bedirhan 将项目拆分为四个独立的 Claude Code 会话（语音引擎、内容生成、3D 游戏层、核心应用），确保每个上下文干净清晰，并行推进。他采用了"口述而非打字"的开发方式，几乎全程通过语音与 Claude 协作。

**商业进展**
项目已获得伊斯坦布尔三所医学院和一家制药公司的关注，将于未来数周内启动试点项目。

**核心建议**
将 Claude 视为"思维伙伴"而非单纯的代码生成器。Bedirhan 原本计划自托管语音引擎，是 Claude 建议他改用云服务商以加快进度——这一建议节省了大量时间。

### 第二名：Wrench Board — Alexis Chapellier（法国·雷尼耶-埃塞里）

**项目背景与痛点**
Alexis 是一位自学成才的电子维修技师，此前已创办过一个面向维修店的 AI 管理平台 RepairMind。他深知独立技术人员在处理复杂电路板故障时，往往缺乏系统性的诊断辅助工具。

**项目功能**
Wrench Board 是一个**智能电路板诊断代理**。用户只需上传电路原理图和板视图（boardview），并描述故障症状，系统便会自动构建统一的电气图谱，推理定位出需要探测的具体焊盘，读取测量结果，并持续更新诊断假设，直至找出故障根因。

**技术亮点**
该项目的核心赌注在于 Opus 4.7 **新增的视觉原理图理解能力**。Alexis 描述了一个关键验证时刻：当他让模型追踪主板上的供电路径时，Wrench Board 的板视图实时亮起，箭头逐步出现，元器件被逐一指认——这一刻让他确信技术路线成立。

**开发方法**
Alexis 在 Claude Design 中完成架构设计和规格文档，再通过内置按钮将项目直接移交 Claude Code 执行。调试阶段采用多代理并行模式，同时运行五至六个代理，每个代理专注一个子领域。

**励志背景**
Alexis 坦言，参加黑客松时他正在寻找"维生工作"。五天内完成这套复杂系统，是他对"自学者同样可以构建雄心勃勃的产品"这一信念的最好诠释。

**核心建议**
深度头脑风暴，并敢于向模型"反推"。当 Claude 表示某些功能在时间内无法完成时，要有勇气说"我还是要试试"——实践证明时间往往是够的。

### 第三名：Maieutic — Paula Vásquez-Henríquez（智利·孔塞普西翁）

**项目背景与痛点**
Paula 是智利德萨罗略大学的计算机科学教师。她观察到，过去两年间，越来越多的学生能通过考试，却对自己提交的代码一无所知。学生们借助 AI 生成可运行的代码，但从未真正学会如何**精确描述问题、在编码前规划方案、批判性地阅读自己的代码**。

**项目理念**
Maieutic 的名称来源于古希腊哲学的"产婆术"（苏格拉底式问答法），其核心理念是：**不给学生答案，而是引导学生自己找到答案**。该工具专注于培养学生的计算思维和问题拆解能力，而非替代这一过程。

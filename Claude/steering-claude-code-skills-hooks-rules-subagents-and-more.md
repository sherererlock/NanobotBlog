## 主要内容

### 七种指令方法概览

Claude Code 提供了以下七种方法来向 AI 传递行为指令，每种方法在**加载时机、压缩行为（Compaction Behavior）和上下文成本**三个维度上各有不同：

#### 1. CLAUDE.md 文件（根目录）
- **加载时机**：会话开始时加载，并在整个会话中持续保留
- **压缩行为**：经过记忆化缓存（Memoized），压缩后重新读取
- **上下文成本**：高（每一行都占用 Token，无论是否相关）
- **适用场景**：构建命令、目录结构、Monorepo 配置、编码规范、团队规范

CLAUDE.md 是项目根目录下的 Markdown 文件，分为两类：
- **始终加载型**：根目录的 CLAUDE.md，会话开始即加载，长会话中不会丢失
- **按需加载型**：子目录中的 CLAUDE.md，仅在 Claude 读取该子目录下的文件时才加载

**关键建议**：保持 CLAUDE.md 不超过 200 行，指定文件负责人，并像审查代码一样审查其变更。在大型 Monorepo 中，每个团队目录应有独立的子目录 CLAUDE.md，开发者可通过 `claudeMdExcludes` 设置跳过不相关团队的文件。

#### 2. Rules（规则）
- **加载时机**：用户级规则在会话开始时加载；路径范围规则仅在匹配文件被访问时加载
- **压缩行为**：压缩后重新注入
- **上下文成本**：中等
- **适用场景**：具体约束或规范（例如：所有 API 处理器必须用 Zod 验证输入）

Rules 是存储在 `.claude/rules/` 目录下的 Markdown 文件，提供细粒度的约束性指令。

#### 3. Skills（技能）
- **加载时机**：会话开始时加载名称和描述，技能被调用时才加载完整内容
- **压缩行为**：已调用的技能在共享预算内重新注入，超出后最旧的被丢弃
- **上下文成本**：低（仅在调用时加载完整内容）
- **适用场景**：程序化工作流（如部署或发布检查清单）

#### 4. Subagents（子智能体）
- **加载时机**：会话开始时加载名称、描述和工具列表；通过 Agent 工具调用时才加载完整内容
- **压缩行为**：仅返回最终消息（摘要+元数据）到主会话
- **上下文成本**：极低（在被调用前对主上下文零成本；在独立上下文窗口中运行）
- **适用场景**：并行任务或需要隔离运行的子任务（如深度搜索、日志分析、依赖审计）

#### 5. Hooks（钩子）
- **加载时机**：在生命周期事件触发时执行
- **压缩行为**：完全绕过压缩机制
- **上下文成本**：低（配置存在于主上下文之外）
- **适用场景**：确定性自动化操作（如运行 Linter、压缩前备份聊天历史、完成后发送 Slack 通知、阻止特定命令）

#### 6. Output Styles（输出风格）
- **加载时机**：会话开始时注入系统提示
- **压缩行为**：永不压缩
- **上下文成本**：高（占用上下文窗口，但会覆盖默认系统提示）
- **适用场景**：重大角色转换（如从代码助手切换为通用助手）

#### 7. Appending the System Prompt（追加系统提示）
- **加载时机**：会话开始时通过 CLI 参数传入
- **压缩行为**：永不压缩，仅适用于当次调用
- **上下文成本**：中等（会话首次请求后缓存）
- **适用场景**：语气、回复长度、格式偏好等轻量级调整

## 关键数据与结论

1. **三维决策框架**：选择指令方法时，需综合考量"何时加载"、"是否跨长会话持久化"和"权重高低"三个维度，没有一种方法适合所有场景。

2. **规模化成本问题**：在共享仓库中，CLAUDE.md 往往因各团队不断追加内容而膨胀，导致每个工程师的每次会话都加载大量无关内容，既浪费 Token 配额，又稀释了关键指令的执行效果。

3. **分层管理建议**：
   - 个人/项目偏好 → 根目录 CLAUDE.md 或 Rules
   - 团队特定规范 → 子目录 CLAUDE.md
   - 组织级安全/合规策略 → 通过 MDM 或配置管理工具集中部署，且个人无法覆盖

4. **子智能体的隔离优势**：Subagents 在独立上下文窗口中运行，仅将最终摘要返回主会话，在需要并行处理或隔离复杂子任务时，是成本最优解。

## 实际意义

### 对个人开发者
- 通过合理分配不同类型的指令，可以显著减少无效 Token 消耗，使 Claude Code 在长时间会话中保持精准响应
- 利用 Skills 管理复杂的部署流程，利用 Hooks 实现代码质量自动化门控，大幅提升开发效率

### 对工程团队
- 七种方法提供了清晰的权责边界：团队级规范、组织级策略和个人偏好可以分层管理，互不干扰
- Monorepo 场景下的细粒度控制（子目录 CLAUDE.md + claudeMdExcludes）解决了大型团队协作中的"指令污染"问题

### 对企业与合规场景
- 组织可通过 MDM 统一部

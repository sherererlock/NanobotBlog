# Claude Code 动态工作流深度解析

## 核心主题

Anthropic 于2026年6月正式推出 **Claude Code 动态工作流（Dynamic Workflows）** 功能。该功能赋予 Claude 在执行任务时自动编写并编排专属多智能体协作框架（harness）的能力，实现了从"通用单一上下文执行"到"动态定制多智能体协作"的重大跨越。

---

## 主要内容

### 一、背景与问题起源

此前，Claude Code 默认的执行框架主要面向编程任务设计，虽然也能处理部分其他类型的任务，但在以下特殊场景中表现有限：

- **研究与信息整合任务**
- **安全漏洞分析**
- **多智能体团队协作**
- **代码评审**

为应对这些场景，Anthropic 工程师不得不在 Claude Code 之上手动构建专用框架。动态工作流的出现，让 Claude 能够**自动完成这一工作**，无需人工介入。

---

### 二、动态工作流的技术原理

动态工作流通过执行一个包含特殊函数的 **JavaScript 文件**来运作，核心机制包括：

- **子智能体的生成与协调**：Claude 可自主调度多个子智能体，每个子智能体拥有独立的上下文窗口和聚焦目标；
- **标准 JavaScript 工具支持**：包括 JSON、Math、Array 等函数，用于数据处理；
- **模型选择权**：Claude 可为不同子智能体指定不同的模型，实现智能层级差异化；
- **工作树隔离**：子智能体可在独立工作树（worktree）中运行，确保任务隔离；
- **断点续传**：若工作流被中断（如用户退出终端），恢复会话后可从中断处继续执行，避免重复劳动。

---

### 三、为何需要动态工作流——三大失效模式

文章清晰揭示了在**单一上下文窗口**中执行长期、复杂任务时，Claude 可能陷入的三种典型失效模式：

| 失效模式 | 描述 |
|---|---|
| **智能体懈怠（Agentic Laziness）** | Claude 在复杂任务完成一部分后提前宣告"任务完成"，例如安全审查只完成了50项中的35项 |
| **自我偏好偏差（Self-preferential Bias）** | Claude 在验证或评判自身结果时，倾向于认可自己的答案，影响客观性 |
| **目标漂移（Goal Drift）** | 在多轮对话后，由于每次摘要都有信息损耗，原始目标中的细节（如边界条件或"禁止做X"的约束）逐渐丢失 |

动态工作流通过**将任务拆分给多个独立子智能体**处理，从根本上缓解这三类问题。

---

### 四、动态工作流 vs. 静态工作流

| 对比维度 | 静态工作流 | 动态工作流 |
|---|---|---|
| 构建方式 | 使用 Claude Agent SDK 或 `claude -p` 手动编排 | Claude 自主生成定制框架 |
| 灵活性 | 较通用，需覆盖各种边界情况 | 针对当前任务量身定制 |
| 适用场景 | 可预测的固定流程 | 复杂、高价值、结构多样的任务 |
| 所需智能层级 | 不依赖高级模型 | 基于 Claude Opus 4.8 的强大推理能力 |

---

### 五、常见工作流设计模式

文章介绍了 Claude 构建工作流时可能采用的若干核心设计模式（可组合使用）：

1. **分类-执行模式（Classify-and-Act）**：先用分类器判断任务类型，再路由至对应的处理智能体；
2. **扇出-合成模式（Fan-out-and-Synthesize）**：将任务拆解为多个子步骤，并行执行后汇总结果——特别适合大规模并行处理场景；
3. 其他模式（文章提及但因内容截取未完整展示）可能包括：验证循环、对抗性评审、锦标赛评选等。

---

### 六、使用方式

- 直接在提示中要求 Claude 创建工作流；
- 使用触发词 **"ultracode"** 可确保 Claude 优先采用工作流模式；
- 工作流可**分享与复用**，便于团队协作。

---

## 关键数据与结论

- 动态工作流功能于 **2026年6月2日**正式发布；
- 当前支持该能力的核心模型为 **Claude Opus 4.8**；
- 文章明确提示：动态工作流**消耗 Token 较多**，最适合处理**复杂、高价值任务**，不建议用于简单操作；
- 最佳实践仍在持续演进中，属于技术前沿能力。

---

## 实际意义

### 对个人开发者
动态工作流极大降低了构建复杂自动化流程的门槛。无需具备多智能体系统的专业知识，用户只需用自然语言描述需求（如"对我的商业计划书进行投资人、客户、竞争对手三方视角的多维拆解审查"），Claude 就能自动生成合适的协作框架并执行。

### 对企业与团队
这一功能为企业高价值场景提供了更强的可靠性保障。例如：大规模代码重构（将 User 模型全局重命名为 Account）、简历批量筛选、Slack 事故日志根因分析等任务，可以以更系统化、更少偏差的方式完成，并支持工作流的跨团队共享与复用。

### 对 AI 工程范式的启示
动态工作流代表了一种 **"元编程"式的 AI 能力演进**——AI 不仅执行任务，还能设计执行任务的系统本身。这标志着 AI 辅助工具正从"单步执行助手"向"自主工程编排者"转变，是通向更高级别 AI 自治能力的重要一步。

> **总结**：Claude Code 动态工作流通过赋予 Claude 自主设计和编排多智能体协作系统的能力，有效解决了单上下文窗口在复杂任务中的固有局限，是 AI 编程工具走向真正自主化任务执行的重要里程碑。

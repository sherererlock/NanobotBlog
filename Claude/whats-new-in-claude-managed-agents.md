# Claude Managed Agents 新功能深度解析：定时调度与环境变量保险库

## 核心主题

2026年6月9日，Anthropic 正式宣布 Claude Managed Agents 平台推出两项重要功能更新，现已进入公开测试阶段：**定时调度运行（Run Agents on a Schedule）** 与 **环境变量保险库存储（Store Environment Variables in Vaults）**。这两项功能标志着 Claude 托管智能体平台在自动化能力和安全基础设施方面迈出了关键一步，旨在帮助企业团队更高效、更安全地部署 AI 智能体，承接真实业务工作流。

---

## 主要内容

### 一、定时调度：让智能体自动完成例行任务

此前，Claude Managed Agents 的运行需要手动触发。新的定时调度功能允许用户为智能体部署配置 **Cron 时间表**，每次计划触发时，智能体会自动开启新会话并独立完成任务，无需用户自行构建或托管任何调度基础设施。

**核心机制：**
- 用户为部署项设置 Cron 计划表
- 每次调度触发时，智能体自动启动独立会话
- 支持随时暂停、恢复、归档或按需手动触发额外运行

**典型应用场景包括：**
- 夜间数据同步
- 每周合规扫描
- 每日摘要生成
- 生产日志与指标监控

**企业实际应用案例：**
- **Rakuten**：利用定时调度，按周或月度频率自动分析电子表格数据并生成报告与演示文稿，同时监控生产日志和应用健康状态，无需额外搭建分析仪表板。
- **Actively AI**：基于 Claude Managed Agents 构建跨账户智能搜索系统，为销售团队提供定期刷新的查询结果（如"今天应联系哪些客户"），并成功以定时调度取代了团队此前自建的调度基础设施，显著简化技术栈、加快产品迭代周期。
- **Ando**：智能体自动监控频道中提出的后续行动，在到期时主动跟进并发送会议提醒，辅助招聘和销售团队保持运转节奏。

---

### 二、环境变量保险库：安全认证 CLI 工具与外部服务

智能体连接外部系统的方式主要包括直接 API 调用、命令行工具（CLI）和 MCP（模型上下文协议）。此次更新将 **保险库（Vaults）** 功能扩展至支持环境变量存储，使 CLI 及其他工具能够以安全方式完成认证请求，而不将真实密钥暴露给模型本身。

**核心安全机制：**
- 用户在保险库中注册 API 密钥，并指定对应的环境变量名称与允许访问的域名
- 智能体沙箱中仅持有"占位符"，真实密钥在**网络边界处**附加，且仅对已授权域名生效
- 更新密钥时，只需在保险库中修改，正在运行的会话将在下次调用时自动获取新值，操作简便且无缝衔接

目前已支持此方式的 CLI 工具包括：**Browserbase、KERNEL、Notion、Ramp、Sentry** 等。值得特别关注的是，Browserbase 和 KERNEL 的接入首次为 Claude Managed Agents 带来了**浏览器操作能力**，使智能体能够在使用其他工具的同时，直接导航和交互网页。

**企业实际应用案例：**
- **Notion**：通过保险库中的环境变量安全部署其 CLI，在 MCP 工具的基础上新增文件上传功能，且 API 令牌全程不暴露给模型，满足安全团队的严格合规要求。
- **Browserbase**：使用 Claude Managed Agents 结合 `browse` CLI，通过保险库认证，构建了公开的浏览器技能目录；同时配合定时调度，定期验证目录准确性。
- **KERNEL**：通过保险库安全连接智能体与其用于追踪使用量和客户对话的数据库，实现实时监控使用量突增并自动标记，辅助团队及时与客户确认异常活动。
- **Milana**：借助保险库中的环境变量，将 AI 产品工程师安全连接至客户代码库，实现自动化 Bug 修复，同时大幅提升了大规模数据分析的速度与输出质量。

## 关键数据与结论

| 维度 | 要点 |
|------|------|
| 发布状态 | 公开测试阶段（Public Beta） |
| 发布时间 | 2026年6月9日 |
| 安全边界 | 真实密钥仅在网络边界附加，智能体沙箱持有占位符 |
| 域名限制 | 密钥仅对用户显式授权的域名有效 |
| 调度方式 | 标准 Cron 表达式，无需自建调度服务 |
| 新增能力 | 首次支持浏览器操作（通过 Browserbase/KERNEL CLI） |

多家企业的反馈均指向一个共同结论：**定时调度降低了基础设施复杂度，环境变量保险库实现了安全与效率的平衡**——两者合力，使智能体从"按需工具"升级为"持续运转的自主工作流节点"。

## 实际意义

**对企业运营而言**，定时调度功能消除了企业团队额外搭建调度系统的工程负担，Actively AI 的案例清晰印证了这一点——他们直接用平台原生调度能力替换了自研基础设施，产品迭代速度随之提升。对于拥有大量例行分析任务（如数据报告、合规检查、日志监控）的企业，这一能力具有显著的降本增效价值。

**对安全合规而言**，环境变量保险库解决了长期以来 AI 智能体场景中的核心痛点：如何在不将敏感凭证暴露给模型的前提下，让智能体调用需要认证的外部服务？Notion 安全团队的认可表明，这一设计达到了企业级合规标准，具备在高监管行业（如金融、医疗）推广的潜力。

**对平台生态而言**，浏览器能力的引入（通过 Browserbase 和 KERNEL CLI）大幅拓宽

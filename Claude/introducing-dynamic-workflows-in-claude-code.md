# Claude Code 动态工作流功能深度解析

## 核心主题

Anthropic 于2026年5月28日正式推出 **Claude Code 动态工作流（Dynamic Workflows）** 功能，这是一项面向复杂、大规模工程任务的突破性能力升级。该功能允许 Claude 自主编写编排脚本，在单次会话中驱动数十乃至数百个并行子智能体协同工作，将原本需要数周甚至数季度完成的工程任务压缩至数天内完成。

---

## 主要内容

### 1. 功能定义与设计逻辑

动态工作流的核心在于"动态编排"——Claude 根据用户提示自主制定计划，将任务拆解为若干子任务，并将这些子任务分配给大量并行运行的子智能体。各智能体从独立视角处理问题，其他智能体负责验证和反驳所得结论，整个流程持续迭代，直至结果收敛。这种机制使得动态工作流能够产出单次推理无法达到的深度结论。

这一设计解决的核心痛点是：**单个智能体在处理大型、复杂或遗留代码库时的能力瓶颈**。例如，跨越整个服务的漏洞排查、涉及数百个文件的迁移工程，或需要在多个维度进行压力测试的架构方案，都超出了单次对话所能覆盖的范围。

### 2. 适用场景

官方列举了动态工作流的三大核心使用场景：

- **全代码库级别的审查与安全审计**：Claude 对整个服务或代码仓库进行并行搜索，独立验证每一个发现，确保报告中只呈现真实问题。这同样适用于安全加固工作，如身份验证检查、输入验证和不安全代码模式的全局排查。

- **大规模迁移与现代化改造**：支持跨越数千个文件的框架替换、API 废弃处理和编程语言移植等端到端任务。

- **高风险任务的多重验证**：当错误代价极高时，工作流可以让多个独立智能体分别尝试解决同一问题，并引入"对抗性智能体"主动寻找结果中的漏洞，在用户看到最终答案前完成内部自检。

### 3. 启动方式

用户有两种方式激活动态工作流：

- **直接指令**：向 Claude 明确发出创建工作流的请求，例如输入"Create a workflow"；
- **Ultracode 模式**：通过 Claude Code 的 Effort 菜单开启专属设置 `ultracode`，该模式会将努力级别设置为 `xhigh`，并允许 Claude 自动判断何时需要调用工作流。

官方建议在使用动态工作流时同步开启**自动模式（auto mode）**，以获得最佳体验。

### 4. 技术架构特点

动态工作流具备若干关键的工程设计特性：

- **进度持久化**：工作流运行过程中持续保存进度，任务中断后可从断点续跑，而非重头开始；
- **会话外协调**：任务编排发生在对话上下文之外，确保计划不受任务规模增长的影响而偏离；
- **结果融合机制**：子智能体的输出在汇总前经过交叉验证，用户最终收到的是统一、协调的单一答案；
- **首次确认提示**：工作流首次触发时，Claude Code 会展示即将执行的内容并请求用户确认；
- **管理员控制**：企业组织的管理员可通过托管设置禁用工作流功能。

### 5. 可用范围

动态工作流当前以**研究预览（Research Preview）**状态发布，支持平台包括：Claude Code CLI、桌面端、VS Code 扩展，以及 Claude API、Amazon Bedrock、Vertex AI 和 Microsoft Foundry。在计划层面，Max、Team 计划默认开启；Enterprise 计划默认关闭，需管理员手动启用。

## 关键数据与案例

### Bun 项目重写：规模验证的里程碑案例

文章重点介绍了动态工作流在真实大型项目中的应用——**Bun 运行时从 Zig 语言移植至 Rust 语言**的完整案例：

| 指标 | 数据 |
|------|------|
| 移植代码规模 | 约 **75 万行 Rust 代码** |
| 测试套件通过率 | **99.8%** |
| 完成时间 | 从第一次提交到合并，仅用 **11 天** |

整个工作流拆解为多个阶段：首先由一个工作流映射 Zig 代码库中每个结构体字段对应的 Rust 生命周期；随后数百个智能体并行为每个 `.zig` 文件编写行为等价的 `.rs` 文件，每个文件配置两名审查智能体；之后启动修复循环驱动构建和测试直至全部通过；移植完成后，另一个隔夜工作流处理不必要的数据拷贝，并为每项优化自动提交 PR。

### 工程师实践反馈

- **Alessio Vallero（高级工程经理）**：动态工作流在大型代码库的发现与审查任务中表现突出，能识别传统静态分析工具遗漏的死代码和清理机会，帮助工程师提升维护和重构效率；
- **Ken Takao（首席系统工程师）**：动态工作流填补了单个子智能体与完整智能体团队之间的能力空白，使从计划到实施的流程更加流畅，长时间运行任务的可信度和可观测性均得到提升。

## 实际意义

### 对软件工程实践的影响

动态工作流代表着 AI 辅助编程从"对话级"向"项目级"的范式跃迁。传统 AI 编程助手擅长处理局部、片段式的代码任务，而动态工作流首次使 AI 具备了接管跨越整个代码库、持续数天的复杂工程项目的能力。这对于维护大型遗留系统、推进架构现代化、执行安全合规审计的工程团队而言，具有显著的生产力提升价值。

# Code w/ Claude London 2026：重新思考我们的构建方式

## 核心主题

本文是 Anthropic 于 2026 年 5 月在伦敦举办的"Code w/ Claude"开发者大会的官方回顾报道。此次活动是继旧金山站之后，Claude 开发者系列活动首次登陆欧洲，核心议题围绕 **AI 如何重塑软件开发范式**，并正式宣布了针对企业级用户的两项重要新功能。

---

## 主要内容

### 一、大会背景与主旨

"Code w/ Claude London 2026"汇聚了来自欧洲及全球的构建者、开发者与创业者，历时两天，涵盖主题演讲、分组讨论与动手工作坊等多种形式。

大会由 Claude Code 产品负责人 **Boris Cherny** 开场致辞。他以自身经历引入主题：中学时代，他曾编写 TI-83 计算器程序来解数学题，也曾自学 HTML 优化 eBay 上的宝可梦卡片销售页面。那种"写完就能跑"的即时反馈感，让他第一次体会到编程的魔力。

然而，随着软件工程日益复杂——编译器、类型检查器、构建系统层层叠加——"我有一个想法"与"它真正运行"之间的距离被不断拉长。Boris Cherny 指出，**AI 智能体正在压缩这段距离**：你描述一个问题，程序便随之呈现。这就像当年的计算器体验，只不过现在这台"计算器"可以写出完整的分布式系统。

这一哲学贯穿整场大会的核心叙事：**AI 代码工具的本质，是在还原编程最初的创造乐趣**。

### 二、技术演示与客户案例

大会期间，Anthropic 团队展示了如何超越 Claude Code 的基础用法，包括如何优化模型的"思考预算"与不同任务下的算力投入策略。

与此同时，多家企业客户分享了实际应用经验，包括：
- **Spotify**：全球领先音乐流媒体平台
- **Base44**：AI 应用构建平台
- **Legora**：面向法律行业的 AI 工具

这些案例共同印证了 AI 编程工具在真实生产环境中重新激活"创造者感受"的可能性。

### 三、重磅产品公告：Claude Managed Agents 新功能

大会最核心的发布内容，是 **Claude Managed Agents（托管智能体）** 的两项新能力，专门面向企业级安全合规需求而设计：

#### 1. 自托管沙箱（Self-hosted Sandboxes，公开测试版）

工具执行环境从 Anthropic 基础设施迁移至**用户自行配置的环境**，可选择企业自有基础设施，或 Cloudflare、Daytona、Modal、Vercel 等托管服务商。

核心优势：
- **数据不出边界**：文件与代码仓库不离开企业私有网络
- **合规可控**：企业自有的网络策略、审计日志与安全工具均可无缝应用
- **算力自定义**：企业可自主控制计算资源规模与运行时镜像
- **职责分离清晰**：智能体的编排循环、上下文管理与错误恢复仍由 Anthropic 基础设施负责，工具执行层则完全在用户掌控之下

#### 2. MCP 隧道（MCP Tunnels，研究预览版）

允许智能体访问企业**私有网络内部的 MCP 服务器**，而无需将其暴露到公共互联网。

技术特点：
- 企业侧仅需部署一个轻量级网关，发起**单条出站连接**
- **无需开放入站防火墙规则**，无需公开服务端点
- 全程流量**端到端加密**
- 通过 Claude Console 由组织管理员统一管理
- 已在 Managed Agents 与 Messages API 中同步支持

## 实际意义

### 对开发者而言

此次大会传递的核心信号是：AI 编程工具正从"辅助工具"演变为**真正的创作伙伴**。通过优化思考预算、拓展智能体能力，Claude Code 正在努力让每一位开发者——无论经验深浅——都能回归编程最本质的乐趣：**快速将想法转化为可运行的现实**。

### 对企业用户而言

自托管沙箱与 MCP 隧道的推出，直接回应了企业在采用 AI 智能体时面临的两大核心顾虑：**数据主权**与**网络安全合规**。企业无需在"AI 能力"与"安全边界"之间二选一，Anthropic 通过架构设计将二者兼顾——智能化由 Anthropic 负责，数据边界由企业掌控。

### 对行业而言

从旧金山到伦敦再到东京，"Code w/ Claude"系列活动的全球化布局，标志着 AI 原生开发工具的竞争已进入**生态与社区建设**阶段。企业级功能的持续完善，意味着 AI 编程助手正加速从开发者玩具迈向生产级基础设施。

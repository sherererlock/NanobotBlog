# Claude Managed Agents 更新深度总结

## 核心主题

Anthropic 于 2026 年 5 月 19 日宣布 Claude Managed Agents 平台迎来重大更新，推出两项关键新功能：**自托管沙箱（Self-hosted Sandboxes）** 和 **MCP 隧道（MCP Tunnels）**。这两项功能的核心价值在于：将 AI 代理的执行环境与数据资产保留在企业自身的安全边界之内，同时维持 Anthropic 托管编排层的可靠性优势，从根本上解决企业级 AI 代理部署中的安全合规痛点。

---

## 主要内容

### 一、自托管沙箱（Self-hosted Sandboxes）

**功能定位：** 允许企业将 AI 代理执行工具的计算环境部署在自己掌控的基础设施上，而非完全依赖 Anthropic 的云端环境。

**架构分工：**
- **Anthropic 侧**：保留代理循环（Agent Loop），负责编排逻辑、上下文管理和错误恢复；
- **企业侧**：负责工具执行的计算环境，包括运行时镜像、资源配置、网络策略和审计日志。

这种"编排在云端、执行在本地"的混合架构，既保留了 Anthropic 托管服务的稳定性，又将敏感数据和计算资源的控制权交还给企业。

**支持的沙箱提供商：**

| 提供商 | 核心特点 |
|--------|----------|
| **Cloudflare** | 微虚拟机（microVM）隔离，零信任密钥注入，可自定义代理审计出站流量，支持通过 Cloudflare 网络访问内部服务 |
| **Daytona** | 长期运行、有状态的完整计算机环境，支持 SSH 访问和状态暂停/恢复，适合数小时的持续任务 |
| **Modal** | 专为 AI 工作负载构建，支持自定义容器运行时，可扩展至数十万并发沙箱，提供按需 CPU/GPU 资源 |
| **Vercel** | 毫秒级启动，结合 VM 安全隔离与 VPC 对等，凭据在网络边界注入而不进入沙箱内部 |

**核心优势：**
- 敏感文件、软件包和服务不离开企业网络边界；
- 原有的网络策略、审计日志、安全工具自然适用；
- 企业可自主控制计算资源规模，满足长编译、图像生成等重型工作负载需求。

**当前状态：** 公开测试版（Public Beta）。

---

### 二、MCP 隧道（MCP Tunnels）

**功能定位：** 允许 Claude 代理连接到企业私有网络内部的 MCP（Model Context Protocol）服务器，无需将内部服务暴露至公共互联网。

**技术原理：**
企业部署一个轻量级网关，该网关向外建立单条出站连接，无需开放任何入站防火墙规则，也不需要创建公开端点，全程流量端对端加密。

**可访问的内部资源类型：**
- 内部数据库
- 私有 API
- 知识库
- 工单系统等企业内部工具

**支持范围：** 兼容 Managed Agents 和 Messages API，由企业组织管理员通过 Claude Console 的工作区设置进行统一管理。

**当前状态：** 研究预览阶段（Research Preview），需申请访问权限。

## 实际意义

### 1. 破解企业 AI 代理部署的安全困局

此前，企业使用云端 AI 代理服务时，面临数据主权与服务能力之间的两难选择：要么接受敏感数据上云的风险，要么放弃托管服务的便利性自行搭建。自托管沙箱的推出为这一困境提供了架构层面的解决方案——编排能力由 Anthropic 维护，执行环境由企业掌控，两者通过明确的接口解耦。

### 2. 降低企业 AI 代理的落地门槛

MCP 隧道允许代理直接调用企业内网的数据库、API 和知识库，无需重新设计网络架构，内部系统的连通成本大幅降低。结合合作伙伴案例中"两天上线""一周内可用"的交付速度，可以判断该方案显著缩短了企业 AI 代理的工程化周期。

### 3. 推动 AI 代理平台生态成熟

通过官方支持 Cloudflare、Daytona、Modal、Vercel 四家主流计算基础设施提供商，Anthropic 构建了一个开放的"编排层+执行层"分离生态，使企业可以基于已有的基础设施投资和技术栈选择，而非被迫采用单一厂商方

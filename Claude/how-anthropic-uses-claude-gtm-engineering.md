# Claude Code 如何帮助一位销售员转型为 GTM 工程师：Anthropic 内部实践深度总结

---

## 核心主题

本文以 Anthropic 销售团队成员 Jared Sires 的真实案例为核心，讲述了一位**从未写过一行代码的销售代表**如何借助 Claude Code，从零开始构建 AI 驱动的销售工作流工具，最终转型为公司 GTM（Go-To-Market，市场进入）产品经理的完整故事。文章的核心信息是：**AI 编程工具正在打破技术门槛，使非技术人员也能成为实际意义上的软件开发者**。

---

## 主要内容

### 一、问题背景：销售工作的"行政困境"

Jared Sires 于 2024 年加入 Anthropic，担任初级客户经理（Account Executive）。随着业务快速扩张，他的账户池迅速膨胀至 **600 至 700 个客户账户**，每天需要接听 **10 至 15 个客户电话**，同时还需处理大量邮件往来。每晚加班到 9 至 10 点回复客户邮件成为常态，这严重压缩了他用于真正销售工作的时间。

此外，Anthropic 产品迭代速度极快，**每 24 至 48 小时就会有产品更新**，涉及 API 限速、定价模型、SDK 行为等技术细节。客户问题往往指向最新版本的文档内容，而销售代表必须在 Slack、Google Docs、内部知识库和开发者文档之间频繁检索——这是一项高度重复且容易出错的任务。

### 二、第一个实验：账户优先级智能排序

Jared 最初的尝试相对保守。他使用 Google Apps Script 与 Claude API 配合，从内部系统拉取产品使用数据，让 Claude 每天早晨根据账户增长速度进行排名，生成"每日简报"，告诉他当天应该重点联系哪些客户。这一机制使他在 700 个账户中快速定位优先目标，有效减少了决策的认知负担。

### 三、核心产品：CLAFTS 的诞生与迭代

**CLAFTS**（Claude dRAFTS 的缩写）是整篇文章的技术核心——一个嵌入 Gmail 的应用程序，利用 Claude API 自动起草客户邮件回复。

**技术构成：**
- 代码量约为 **4,300 行**，几乎全部由 Claude Code 编写
- 连接共享 Google Drive 文件夹获取内部上下文
- 通过网络搜索实时引用 Anthropic 官网最新文档
- 基于系统提示（System Prompt）深度匹配 Jared 本人的写作风格

**迭代过程：**
Jared 经历了**数百次系统提示迭代**，才将 Claude 的默认写作风格（偏长、措辞保守）调整为符合自己习惯的简洁风格。他进一步开发了 **CLAFTS Tones** 功能，通过模式匹配识别不同关系类型（客户、同事、家人）并相应调整邮件语气。

一个有趣的测试细节：Jared 用私人账号给自己发送了一系列"语气越来越愤怒"的邮件来测试该功能。当 Claude 开始模仿愤怒语气，但随后**拒绝继续生成充满敌意的客户邮件**时，他意识到功能调试成功——Claude 的内置价值观在实际产品中发挥了边界作用。

### 四、从工具使用者到产品构建者

CLAFTS 在 Slack 上分享后，**24 小时内被销售组织内多名同事采用**，产生了相似的效率提升效果。这一反响推动了 Jared 职业角色的正式转变：他被任命为 **GTM 团队产品经理**，专职识别销售运营痛点并构建 Claude 驱动的解决方案。

目前，他构建的工具涵盖：
- 自动化客户沟通系统
- 电话前的客户背景研究工具
- 基于会议录音自动生成跟进邮件的功能

所有工具均被打包为 **Claude Cowork 插件**，供整个销售团队调用。

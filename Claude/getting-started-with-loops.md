# 深度总结：《循环入门指南》——Claude Code 的智能体循环设计

## 核心主题

本文由 Anthropic 的 Claude Code 团队撰写，系统阐述了"智能体循环"（Agentic Loops）的概念、分类及实践指南。文章的核心观点是：**与其单纯地"提示"编程智能体，不如学会"设计循环"**——即让智能体在满足终止条件之前，持续重复执行工作周期。这一思路代表了 AI 辅助编程从被动响应向主动执行的范式转变。

---

## 主要内容

### 一、什么是循环？

Claude Code 团队对"循环"给出了明确定义：**智能体重复执行工作周期，直至满足停止条件**。这一定义打破了社区中对"循环"概念众说纷纭的混乱局面，提供了清晰的框架。

循环的分类维度包括四个方面：
- **触发方式**（如何启动）
- **停止标准**（何时结束）
- **使用的 Claude Code 原语**（底层机制）
- **适用任务类型**（最佳使用场景）

文章特别强调：**并非所有任务都需要复杂循环**，应从最简单的解决方案出发，按需选用不同模式。

---

### 二、四种循环类型详解

#### 1. 基于轮次的循环（Turn-based Loops）
- **触发方式**：用户手动提示
- **停止标准**：Claude 判断任务完成，或需要额外信息
- **适用场景**：较短的、非定期的一次性任务
- **典型示例**：让 Claude 创建一个"点赞按钮"，它自主读取代码、完成编辑、运行测试，再返回结果

这是最基础的循环形态，本质上是"每次提示即启动一轮智能体循环"。文章特别介绍了通过编写 **SKILL.md** 文件来提升 Claude 自我验证能力的方法——将人工检查步骤编码为结构化指令，让 Claude 像人类审查员一样进行端到端验证（包括启动开发服务器、截图前后对比、检查控制台错误、运行性能追踪等），从而减少对人工介入的依赖。

#### 2. 基于目标的循环（Goal-based Loops，使用 `/goal`）
- **触发方式**：实时手动提示
- **停止标准**：目标达成，或达到最大轮次上限
- **适用场景**：具有可验证退出条件的复杂任务
- **典型示例**：`/goal 将首页 Lighthouse 评分提升至 90 分以上，最多尝试 5 次`

该模式引入了**评估者模型**（Evaluator Model）机制：每当 Claude 准备停止时，评估者会检查成功标准是否满足，若未达到则将其"推回"继续工作。这解决了单轮循环中 Claude 自行判断"够好"而过早退出的问题。文章强调，**确定性标准**（如测试通过数量、分数阈值）效果最佳，因为它们消除了主观判断的模糊性。

#### 3. 基于时间的循环（Time-based Loops，使用 `/loop` 和 `/schedule`）
- **触发方式**：指定时间间隔
- **停止标准**：用户手动取消，或任务自然完成（如 PR 合并、队列清空）
- **适用场景**：定期重复的工作，或需要与外部系统交互的任务
- **典型示例**：`/loop 5m 检查我的 PR，处理评审意见并修复失败的 CI`

`/loop` 在本地计算机运行，关机即停止；`/schedule` 则可将循环迁移至云端，实现持久化执行。这一类型适合处理"任务内容不变、但输入随时间变化"的场景，如每日汇总 Slack 消息、持续监控 PR 状态等。

#### 4. 主动式循环（Proactive Loops）
- **触发方式**：事件或定时触发，无需实时人工干预
- **停止标准**：每个子任务在目标达成时退出；整体例程持续运行直至用户关闭
- **适用场景**：高频、定义明确的重复性工作流，如 Bug 报告处理、Issue 分类、代码迁移、依赖升级等
- **资源管理**：将例程路由至更小、更快的模型，仅在需要判断时调用最强模型

主动式循环是最高级的形态，通过组合 `/schedule`、`/goal`、自动模式和动态工作流等多种原语，构建出**全自动、长期运行的工作管线**。例如，利用 `/schedule` 定期检查新反馈报告，配合 `/goal` 定义"完成"的标准，实现无人值守的持续处理。

## 关键数据与结论

| 循环类型 | 触发方式 | 停止机制 | 适用任务复杂度 |
|----------|----------|----------|--------------|
| 基于轮次 | 手动提示 | Claude 自判断 | 低 |
| 基于目标 | 手动提示 | 评估者模型 + 轮次上限 | 中 |
| 基于时间 | 时间间隔 | 手动取消/任务完成 | 中高 |
| 主动式 | 事件/定时 | 子任务目标达成 | 高 |

核心结论：
- **确定性退出标准优于主观判断**：数值型、可量化的成功标准能显著提升循环效率
- **Token 管理是设计循环的重要约束**：每种循环类型都附带相应的 Token 控制建议
- **复杂度递进原则**：从最简单的轮次循环起步，按需升级，避免过度工程化

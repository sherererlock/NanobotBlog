# Claude 计算机与浏览器使用最佳实践：深度总结

## 核心主题

本文是 Anthropic 官方发布的技术指南，面向正在将 Claude 的**计算机使用（Computer Use）**与**浏览器使用（Browser Use）**能力集成到产品中的开发者。文章从分辨率配置、工具选择、提示词设计、智能体行为规范等多个维度提供了系统性的最佳实践建议，帮助开发者构建更稳定、更准确的 AI 自动化系统。适用模型为 Claude 4.6 系列（Opus 4.6、Sonnet 4.6、Haiku 4.5）以及 Claude Opus 4.7。

---

## 主要内容

### 一、分辨率与缩放配置（最高优先级优化）

点击精度是计算机使用集成的基础。文章指出，**在将截图发送至 API 之前预先降低分辨率**是单一效益最高的优化手段。

**核心问题根源**：Claude 的 Computer Use API 在内部对超出限制的图像进行静默降缩处理。若开发者发送原始高分辨率截图，模型看到的是降级后的图像，但坐标系仍对齐原始分辨率，从而导致点击位置偏移。

**各模型的 API 图像限制：**
- **Claude 4.6 系列**：最长边不超过 1568 像素，总像素不超过 115 万（1.15 MP）
- **Claude Opus 4.7**：最长边不超过 2576 像素，总像素不超过 375 万（3.75 MP）

**推荐分辨率：**
- **通用默认值**：1280×720（720p），适用于绝大多数场景，占用约 80% 的像素预算，稳定性好
- **Opus 4.7 推荐**：1080p 起步，质量显著优于 720p
- **进阶方案**："最大 API 适配"算法，根据原始图像宽高比动态计算最优分辨率，避免比例失真，充分利用像素预算

**需要避免的分辨率：**
- 原始未缩放截图（最常见的点击精度问题根源）
- 低于 960×540 的超低分辨率（细节损失过多）
- macOS 上需注意设备像素比为 2 的问题，实际截图分辨率可能是屏幕坐标的两倍
- 4.6 系列应避免 1920×1080 及以上（超出像素限制会被静默降缩）

文章还提供了坐标换算的注意事项：模型返回的坐标基于发送图像的分辨率，需将其按比例映射回实际屏幕坐标。

### 二、工具选择策略

文章区分了两种主要工具类型：

- **计算机使用工具（Computer Use Tool）**：截图 + 鼠标/键盘操作，通用性强，适用于任何桌面环境
- **浏览器使用工具（Browser Use Tool）**：专为 Web 自动化设计，直接与 DOM 交互，无需依赖截图，精度更高、速度更快、成本更低

**核心建议**：当任务场景明确为 Web 浏览器操作时，**优先使用浏览器使用工具**，而非截图方式的计算机使用工具。浏览器工具可直接点击 DOM 元素，不受视觉分辨率限制，稳定性更强。

此外，文章建议开发者**为 Claude 提供专用的辅助工具**，例如：
- 从剪贴板读取内容的工具
- 获取当前光标位置的工具
- 专用的截图工具（区分于通用计算机工具）

这些辅助工具可减少模型的自主判断负担，提高任务成功率。

### 三、提示词工程与任务规划

**系统提示词设计原则：**
- 明确告知模型当前的操作环境（操作系统、浏览器类型、屏幕分辨率）
- 描述任务的整体背景与目标
- 提供任务完成的判断标准，指导模型何时应停止并汇报结果

**任务分解建议：**
- 对于复杂的多步骤任务，建议将其拆分为明确的子任务序列
- 避免一次性给予过于模糊或过于宏大的目标
- 适时引入"检查点"机制，让模型在关键步骤前确认状态

**错误处理建议：**
- 指示模型在遇到意外情况时主动截图并汇报，而非盲目重试
- 设定最大重试次数，防止模型陷入无限循环

---

### 四、智能体行为规范与安全边界

文章特别强调了在构建自动化智能体时的**安全性与可控性**：

- **最小权限原则**：仅授予模型完成任务所需的最低系统权限
- **人工确认节点**：对于涉及不可逆操作（如提交表单、删除文件、发送邮件）的步骤，建议在执行前引入人工确认环节
- **提示注入防护**：浏览器环境中网页内容可能包含恶意指令，需在系统提示中明确告知模型忽略来自页面内容的指令
- **会话隔离**：每次任务应使用独立的浏览器上下文，避免 Cookie 或登录状态污染

## 关键数据与结论

| 指标 | Claude 4.6 系列 | Claude Opus 4.7 |
|------|----------------|----------------|
| 最长边限制 | 1568 像素 | 2576 像素 |
| 总像素限制 | 1.15 MP | 3.75 MP |
| 推荐默认分辨率 | 1280×720 | 1920×1080 |

- 超出分辨率限制是点击精度下降的**首要原因**，解决后效果优于几乎所有其他优化手段
- 在 Web 场景中，浏览器使用工具相比截图方式在**速度、精度和成本**上均有显著优势
- 720p 分辨率使用约 **80%** 的像素预算，是安全性与性能的最优平衡点
